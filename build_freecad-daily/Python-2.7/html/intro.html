IntroductionÂ¶

This document illustrates several FreeCAD scripting examples. It also uses functions from a module testEqual to help automatically verify that the examples work. The verification is based on the observation that a.cut(b) and b.cut(a) will both be empty if the objects are equal. Based on this, scripts can be used to build an object in different ways, and the results compared to ensure the different approaches actually give the same object.

The test functions are documented in an appendix. The main test function areEqual(a,b) returns a boolean result. Wrapper functions testEqual() and testNotEqual() provide a simpler mechanism for running automatic checks. They do not return a result, but raise an exception if the test fails.

One caveat in testing is that assignment in FreeCAD makes a new pointer to the same object, so manipulation of a second object made this way will also alter the original. That is

a = Part.makeBox(1,100,10)
b = a
a.translate(Vector(0,0,10))
print(areEqual(a, b))

True

Returns True because the object pointed to by both a and b is translated, whereas

a = Part.makeBox(1,100,10)
b = a.copy()
a.translate(Vector(0,0,10))
print(areEqual(a, b))

False

Returns False because the translation of b does not affect a.

To run examples in this document the path of the freecad and testEqual module libraries is needed. This should be done before starting python with export PYTHONPATH=$PYTHONPATH:/usr/lib/freecad/lib:/path/to/testEqual. It can also be done inside a python session using sys.path.append().

Sections of this document use doctest testcode and testoutput directives. If there is any test output then it is displayed in a block below the testcode. All tests use the setup code specified in Spinx conf.py doctest_global_setup variable, which does

import FreeCAD
from FreeCAD import Base, Vector
import Part, Mesh, MeshPart
from testEqual import *

and also set vectors for the origin and directions X, Y, and Z, which are used in several examples.

O = Vector(0,0,0)
X = Vector(1,0,0)
Y = Vector(0,1,0)
Z = Vector(0,0,1)

Examples can be run in a FreeCAD GUI session by cut-and-paste into the console. The testEqual module is not needed except to run tests, which are mainly for the purpose of confirming that the examples in this document continue to work in new FreeCAD versions. However, the O, X, Y, Z will be needed in several examples.
